## 二、中等级别（有明显副作用的逻辑缺陷）

### 6. `web_app.py` + `index.html`：`File.path` 在标准浏览器中不可用

| 项目 | 内容 |
|------|------|
| **文件** | `代码脚本/templates/index.html` |
| **定位** | `original_path: currentFile.path \|\| ''` |
| **现象** | `File.path` 仅存在于 Electron/Chrome 扩展等特定环境中。标准浏览器（Firefox、Safari、标准 Chrome）不提供此属性，`currentFile.path` 始终为 `undefined`。传递给 Flask 后端时 `original_path` 为空字符串，导致 `os.path.abspath("")` 解析到不可预期的目录。报告文件被保存到当前工作目录的父目录（例如 `/Users/Eno/GitHub/`）而非项目目录。 |
| **修复** | 移除对 `File.path` 的依赖，将报告保存到 `_uploads` 目录或让用户指定路径，并提供真正可用的下载机制（如 Flask 的 `send_file` 下载端点）。 |

### 7. `index.html`：`file://` 下载链接在浏览器中不可用

| 项目 | 内容 |
|------|------|
| **位置** | `href="file://${data.report_path}"` |
| **现象** | 主流浏览器禁止从 HTTP(S) 页面打开 `file://` 链接。用户点击"打开报告"无任何反应。 |
| **修复** | 改用 `/api/download?path=...` 的 Flask 端点，用 `send_file` 提供下载。 |

### 8. `run.py`：`--account` CLI 参数被交互提示覆盖

| 项目 | 内容 |

| **文件** | `代码脚本/run.py` |
| **定位** | `main()` 尾部，`if args.serial_no: pass else: account, password = prompt_account()` |
| **现象** | 即使传了 `--account xxx`，只要没有同时传 `--serial-no`，就会调用 `prompt_account()` 用交互输入覆盖掉 `--account` 的值。`--account` 参数形同虚设。 |

### 9. `tester.py` + `web_app.py`：登录失败后继续执行测试

| 项目 | 内容 |
|------|------|
| **文件** | `代码脚本/src/tester.py` |
| **位置** | `run()` 方法登录后 |

### 10. `web_app.py`：`_tasks` 字典无限增长（内存泄漏）

## 三、中低级别（健壮性与设计问题）

### 11. `rate_parser.py`：年龄列搜索上限 30 列

```python
max_col = min(ws.max_column, 30)
```

如果费率表的年龄列在第 31 列之后，搜索直接返回 None，解析失败。同样的 `_find_rate_columns` 限制为 50 列。建议根据费率表实际列数动态确定上限，或允许多个候选列。

### 12. `rate_parser.py`：年龄序列起始条件硬编码 ≤ 1 岁

```python
if a is not None and a <= 1:
```

纯成人产品（年龄从 18 岁起）的费率表永远找不到年龄序列，解析器直接失效。建议改为：先尝试 ≤1 的精确匹配，如果找不到再放宽到最小的年龄值序列。

### 13. `rate_parser.py`：`_read_metadata` 与 `_extract_metadata` 的 data_type 互相覆盖

`_read_metadata` 从产品信息 Sheet 读取精确的 `data_type` 码值（如 `"1"`），但随后 `_extract_metadata` 用正则 `每\d+元.*保险费` 从表头文本重新判断并覆盖前者。如果产品信息 Sheet 明确标注了 data_type，应该优先使用该值，表头文本的猜测仅作为后备。

### 14. `tester.py`：期望费率的浮点精度丢失

```python
@staticmethod
def _float(ws, row, col):
    v = ws.cell(row=row, column=col).value
    if v is None: return None
    try: return float(v)
    except: return None
```

对于保险行业常见的万分位费率（如 0.00042），Excel 中存储为浮点数，经 openpyxl 读取后可能有微小偏差（如 0.0004200000001），导致 PASS/FAIL 判断边界不稳定。建议在比较时使用 `Decimal` 或提高 `tolerance` 的容差。

### 15. `ruitai.yaml`：duty `"3"` 的 `default_value` 类型不一致

```yaml
"3": { name: "身故额外关爱保险金", order: 115219, default_value: ["3"] }
```

其他 duty 的 `default_value` 是字符串（`"2"`、`"4"`），而 duty `"3"` 是数组 `["3"]`。序列化为 JSON 后格式与其他责任不同，可能被 API 拒绝或解析异常。建议统一为字符串。

### 16. `tester.py`：重试循环中 `cookies` 变量名重影

```python
ok, cookies, status, text = api.save_customer(age, gender)
# ...
ok, fee, status, text, reason = api.save_product(
    ..., cookies=cookies, ...
)
```

变量 `cookies` 在同一个作用域内先被赋值为 `save_customer` 的返回值，然后作为参数传给 `save_product`。虽然目前逻辑正确（cookie 应在一次测试流程中保持不变），但变量名重影使代码容易误解。建议在重试循环外捕获 cookies 并用不同名称。

### 17. `run_tests.py`：确认交互在无 `--serial-no` 时才能执行

| 项目 | 内容 |
|------|------|
| **文件** | `代码脚本/run_tests.py` |
| **现象** | `if not args.serial_no:` 控制确认提示和交互输入，但 CLI 模式下没有 `--serial-no` 时会停下等待用户确认。无法在完全非交互的 CI 场景中使用。 |
| **改进** | 增加 `--yes` 或 `--no-confirm` 参数跳过确认。 |

## 四、信息性观察（非缺陷，但值得注意）
